# fwdns

მცირე ზომის DNS ქეშირების სერვერის იმპლემენტაცია Go პროგრამირების ენაზე.

`fwdns` არის შუამავალი სერვერი კლიენტსა და DNS სერვერს შორის, რომელიც
მიიღებს ქსელურ მოთხოვნებს, შეამოწმებს ჩანაწერების არსებობას ლოკალურ
ქეშში და მისი პოვნის შემთხვევაში დააბრუნებს შესაბამის მნიშვნელობას.
ჩანაწერის არ არსებობის შემთხვევაში, მოთხოვნა ზემდგომ DNS სერვერთან გადამისამართდება.

## ფუნქციონალი

- **Caching forwarder** - განმეორებადი DNS მოთხოვნები ქეშში შეინახება, TTL-ისა და მოთხოვნის ტიპის გათვალისწინებით. 
- **Multiple upstreams with failover** - რამდენიმე ზემდგომი DNS სერვერის არსებობის შემთხვევაში, თუ რომელიმე მათგანი მიუწვდომელი იქნება, მოთხოვნა შემდეგ სერვერზე გადავა.
- **Automatic cache cleanup** - უკანა ფონზე ეშვება ფუნქცია, რომელიც ავტომატურად ასუფთავებს ქეშიდან არასაჭირო ჩანაწერებს.
- **Live dashboard** - ვებ-ინტერფეისზე რეალურ დროში ნაჩვენებია DNS მოთხოვნების სტატისტიკა.
- **Single static binary** - ყველა დამატებითი ფაილი, როგორიცაა ვებ-გვერდისთვის საჭირო დოკუმენტები, ერთ გამშვებ პროგრამაშია თავმოყრილი.

## სისტემური მოთხოვნები და გაშვების ინსტრუქცია

#### Makefile

> საჭიროებს make და go 1.26+ პროგრამებს.

```sh
make build         # აგენერირებს ./fwdns გამშვებ ფაილს
make build-windows # აგენერეირებს ./fwdns.exe გამშვებ ფაილს ვინდოუს სისტემისთვის
make run           # აგენერირებს ./fwdns გამშვებ ფაილს და სტარტავს სერვერს
make clean         # ასუფთავებს დაგენერირებულ გამშვებ ფაილებს
```

#### Go

> საჭიროებს მხოლოდ go 1.26+ პროგრამას.

```sh
go build -o fwdns .
```

#### Docker

> საჭიროებს დოკერს.

```sh
docker build -t fwdns .
docker run --rm -p 53:53/udp -p 8080:8080 fwdns
```
